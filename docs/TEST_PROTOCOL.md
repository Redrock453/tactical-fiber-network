# TFN Test Protocol v2.0 — Польові тести з метриками

## Огляд

Цей протокол описує послідовність тестів для перевірки працездатності TFN у польових умовах. Кожен тест має чіткі критерії проходження (pass/fail).

---

## Test 1: Базова зв'язність (Connectivity)

### Обладнання
- 2× медіаконвертери з SFP
- Відрізок відпрацьованого БПЛА-кабелю (1-10 км)
- 2× ноутбуки або Raspberry Pi
- Набір для сплайсингу

### Кроки

| # | Дія | Час | Відповідальний |
|---|-----|-----|----------------|
| 1 | Зачистити обидва кінці волокна | 60 сек | Інженер A+B |
| 2 | Встановити швидкі SC-коннектори | 60 сек | Інженер A+B |
| 3 | Підключити патч-корди до SFP | 30 сек | Інженер A+B |
| 4 | Перевірити LED-індикатори SFP | 10 сек | Обидва |
| 5 | Зчитати RX Power | 30 сек | Інженер A |
| 6 | Запустити ping-тест | 30 сек | Обидва |
| 7 | Запустити iperf3 | 60 сек | Обидва |

### Метрики

| Метрика | Ціль (Pass) | Поріг (Warning) | Fail |
|---------|-------------|-----------------|------|
| RX Power | > -20 dBm | -20 до -25 dBm | < -25 dBm |
| TX Power | > -5 dBm | -5 до -10 dBm | < -10 dBm |
| Link UP | < 30 сек | 30-60 сек | > 60 сек |
| Ping latency | < 1 ms | 1-5 ms | > 5 ms |
| Packet loss | 0% | 0-0.1% | > 0.1% |
| Throughput | > 900 Mbps | 500-900 Mbps | < 500 Mbps |
| BER | 0 | < 10⁻⁹ | > 10⁻⁶ |

### Результат

## Test 2: Механічний сплайс (Splice Quality)

### Обладнання
- Працююча лінія (Test 1 пройдено)
- Механічні сплайси (3-5 шт)
- Стриппер + сколом

### Кроки

| # | Дія | Час |
|---|-----|-----|
| 1 | Розрізати волокно | 5 сек |
| 2 | Зачистити обидва кінці | 20 сек |
| 3 | Сколоти обидва кінці | 15 сек |
| 4 | Встановити механічний сплайс | 15 сек |
| 5 | Виміряти втрати (до/після) | 30 сек |
| 6 | Повторити для 3 сплайсів | |

### Метрики

| Метрика | Pass | Warning | Fail |
|---------|------|---------|------|
| Час встановлення (1 сплайс) | < 45 сек | 45-90 сек | > 90 сек |
| Вставні втрати (1 сплайс) | < 0.3 dB | 0.3-0.5 dB | > 0.5 dB |
| Сумарні втрати (3 сплайси) | < 1.0 dB | 1.0-2.0 dB | > 2.0 dB |
| Збереження лінку | Так | Так з помилками | Лінк впав |
| BER після сплайсингу | 0 | < 10⁻⁸ | > 10⁻⁶ |

## Test 3: DAS-чутливість (Acoustic Detection)

### Обладнання
- Працююча лінія з φ-OTDR або SFP з DOM
- Людина (тестова ціль)
- Автомобіль (тестова ціль, за можливості)

### Кроки

| # | Дія | Відстань від кабелю |
|---|-----|---------------------|
| 1 | Запустити моніторинг DOM/SFP | — |
| 2 | Людина йде поруч із кабелем | 3 м |
| 3 | Людина йде поруч із кабелем | 10 м |
| 4 | Людина йде поруч із кабелем | 20 м |
| 5 | Автомобіль проїжджає поруч | 10 м |
| 6 | Автомобіль проїжджає поруч | 30 м |
| 7 | Стук по кабелю (імітація) | 0 м |

### Метрики

| Метрика | Pass | Warning | Fail |
|---------|------|---------|------|
| Пішохід 3м | Виявлено | Частково | Не виявлено |
| Пішохід 10м | Виявлено | Частково | Не виявлено |
| Автомобіль 10м | Виявлено | Частково | Не виявлено |
| Автомобіль 30м | Виявлено | Частково | Не виявлено |
| Стук (0м) | Виявлено + локалізовано | Виявлено | Не виявлено |
| Хибні спрацьовування | < 1/год | 1-5/год | > 5/год |

## Test 5: Обрив та відновлення (Mesh Failover)

### Обладнання
- Працююча mesh-мережа (3+ вузлів)
- Ножиці (для обриву)

### Кроки

| # | Дія |
|---|-----|
| 1 | Перевірити маршрут між двома вузлами |
| 2 | Обрізати один fiber-лінк |
| 3 | Зафіксувати час виявлення обриву |
| 4 | Зафіксувати час відновлення маршруту |
| 5 | Перевірити новий маршрут |
| 6 | Відновити обрив (механічний сплайс) |
| 7 | Перевірити повернення до оптимального маршруту |

### Метрики

| Метрика | Pass | Warning | Fail |
|---------|------|---------|------|
| Виявлення обриву | < 10 сек | 10-30 сек | > 30 сек |
| Відновлення маршруту | < 30 сек | 30-120 сек | > 120 сек |
| Пакетні втрати при обриві | < 10 | 10-100 | > 100 |
| Повернення до оптимального | Автоматично | Вручну | Не відбулось |
