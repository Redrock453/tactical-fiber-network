# Протокол тестування SpiderLink (v.0.1)

## Тест 1: Пробивна здатність (Connectivity)

### Обладнання
- 2× медіаконвертери з SFP
- Ноутбук (HP EliteBook)
- Відпрацьований кабель БПЛА (5–10 км)

### Що робити
1. З'єднай вузли через волокно
2. Підключи ноутбук до кожного медіаконвертера
3. Зчитуй рівень RX_power через ethtool або веб-інтерфейс

### Критична точка
- Втрати > -30 дБм — зв'язок ляже
- Потрібно зафіксувати, наскільки "брудне" волокно погіршує бюджет лінії

### Очікуваний результат
```
RX Power: -22 dBm ✓
Link: UP
 BER: 0
```

## Тест 2: RF-чутливість (Kerr Effect Test)

### Обладнання
- Радіостанція (Motorola / Baofeng) потужністю 5 Вт
- Волокно (zmotтана котушка)
- Медіаконвертери з SFP

### Що робити
1. Поклади антену рації впритул до волокна
2. Натисни тангенту (передача)
3. Спостерігай за RX_power та BER

### Очікуваний результат
- Під час передачі сигнал "скаче" або з'являються помилки
- Це означає, що волокно працює як сенсор радіополя

### Фізика
Потужне RF-поле змінює показник заломлення скла (ефект Керра), що викликає фазовий зсув світлової хвилі.

---

## Тест 3: Трофейна розвідка (DAS Simulation)

### Обладнання
- Волокно з одним підключеним кінцем
- Другий кінець — обірваний (лежить у "ворожому" секторі)

### Що робити
1. Постукай по кабелю
2. Або проїхай машиною поруч
3. Фіксуй коливання зворотного сигналу

### Очікуваний результат
Якщо SFP фіксує коливання Backscatter — ідея "прослуховування" через ворожі дроти робоча.

```bash
# Моніторинг потужності на MikroTik
/interface ethernet monitor [find where name="sfp1"]
```

```

   rx-power: -22.0dBm
   tx-power: 3.0dBm
   temperature: 45C
```
