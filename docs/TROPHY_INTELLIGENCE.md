# TFN — Трофейна розвідка (Trophy Intelligence)

## Концепція

**Трофейна розвідка** — це підключення до **оптоволоконних ліній зв'язку противника** для отримання розвідданих без виявлення.

Якщо противник використовує оптоволокно (а в умовах РЕБ це неминуче), його кабель стає **вашим сенсором**.

---

## 1. Ситуація на полі бою

### 1.1 Чому у противника є оптоволокно

В умовах потужного РЕБ:
- Радіозв'язок глушиться повністю
- FPV-дрони переходять на волоконне керування (кабель-котушка)
- Польові телефони замінюються оптичними лініями
- З'являються оптоволоконні «нитки» між позиціями

**Результат:** На території противника лежать кілометри оптоволокна.

### 1.2 Типи ворожих ліній

| Тип | Опис | Частота зустрічі | Цінність |
|-----|------|-----------------|----------|
| **FPV-кабель** | Одномодове волокно від дронів | Дуже часто | Низька (тонке) |
| **Польовий зв'язок** | Кабель між позиціями | Часто | Висока |
| **Штабна мережа** | Багатомодове/одномодове до штабу | Рідко | Дуже висока |
| **Цивільна інфраструктура** | Залишені міські мережі | У містах | Середня |

## 2. Методи підключення

### 2.1 Виявлення ворожого кабелю

**Ознаки:**
- Чорна/сіра/жовта оболонка, лежить на землі або в неглибокій траншеї
- Тонкий (1-3 мм) або товстий (5-10 мм) кабель
- Йде від позицій противника
- Може бути маркований (а може й ні)

**Інструменти пошуку:**
- Візуальне спостереження (бінокль, тепловізор вночі)
- Денний патруль у «сірій зоні»
- БПЛА-розвідка (кабель видно з повітря)
- Звіти про FPV-вильоти (де падали — там кабель)

### 2.2 Підключення до обривка

**Сценарій:** Артилерія перебила ворожий кабель. Наш патруль знаходить обривок на нейтральній смузі.

**Дії (3-5 хвилин):**

```
1. Знайти обривок (обидва кінці)
2. Очистити кінець від бруду (серветка)
3. Зачистити буфер стрипером (20 мм)
4. Протерти спиртом
5. Сколоти на 10 мм
6. Встановити швидкий FC-коннектор
7. Підключити до φ-OTDR або міні-OTDR
8. Зчитувати дані
```

**Важливо:** Підключаємося до кінця, який йде **у бік противника**. Цей кінець веде до його позицій і буде «чути» його активність.

### 2.3 Підключення без обриву (TAP)

**Сценарій:** Знайдено цілий працюючий кабель противника.

**Метод 1: Згин (Bend Tap)**
```
1. Обережно зігнути кабель під малим радіусом
2. Частина світла «витече» через згин
3. Детектор поруч фіксує витік
4. Втрати: 0.5-3 дБ (противник може помітити!)
```

**Метод 2: Splitter (Оптичний відгалужувач)**
```
1. Розрізати кабель (викличе обрив у противника!)
2. Вставити 90/10 спліттер (90% — противнику, 10% — нам)
3. З'єднати механічними сплайсами
4. Втрати для противника: 0.5-1 дБ (може не помітити)
5. Час: 5-10 хвилин
```

**Метод 3: Розташований поруч кабель (Coupling)**
```
1. Прокласти наш кабель паралельно ворожому на 5-20 см
2. Акустична связь між кабелями
3. Наш DAS «чути» вібрації на ворожому кабелі
4. Безконтактний метод — виявлення неможливе
```

### 2.4 Таблиця методів

| Метод | Виявлюваність | Якість даних | Час | Складність |
|-------|--------------|-------------|-----|-----------|
| Обривок + OTDR | Низька | Відмінна | 3-5 хв | Низька |
| Bend Tap | Середня | Погана | 30 сек | Низька |
| Splitter TAP | Середня | Добра | 10 хв | Середня |
| Паралельний кабель | Нульова | Середня | 30 хв | Середня |
| Повне підключення | Низька | Відмінна | 15 хв | Висока |

## 3. Що можна дізнатися

### 3.1 DAS на ворожому кабелі

Підключивши φ-OTDR до ворожого кабелю, отримуємо:

```
Наш кінець ←──── Ворожий кабель ────→ Ворожа позиція
                     │
                     ├── 500м: Тиша (нейтральна смуга)
                     ├── 1500м: Пішохід (патруль?)
                     ├── 2000м: Техніка ×2 (БТР?)
                     ├── 2800м: Копання (фортифікація?)
                     ├── 3200м: Генератор 50Гц (командний пункт?)
                     ├── 3800м: Група піхоти (15+ осіб)
                     └── 4200м: Кінець кабелю (або обрив)
```

### 3.2 Перехоплення даних

Якщо ворог передає IP-трафік по оптоволокну:
- Підключення через Splitter → мережева карта → Wireshark
- Аналіз незашифрованого трафіку (якщо немає VPN)
- Навіть при VPN — аналіз метаданих (обсяг, час, напрямок)

### 3.3 Визначення топології ворожої мережі

За OTDR-трасуванням:
- Відстань між вузлами
- Кількість з'єднань (сплайсів)
- Місця розгалужень (спліттери)
- Якість лінії (→ рівень підготовки противника)

### 3.4 Виявлення штабів та КП

Ознаки за DAS-даними:
- Концентрація техніки
- Генератор (50 Гц сигнал)
- Великий обсяг IP-трафіку
- Нічна активність (зміна, наради)

## 6. Контрзаходи противника (і наш захист)

### 6.1 Що може зробити противник

| Дія | Складність | Ефективність |
|-----|-----------|--------------|
| Регулярна перевірка кабелю | Висока (під вогнем) | Низька |
| OTDR-моніторинг своєї лінії | Середня | Висока (виявить TAP) |
| Шифрування трафіку | Середня | Блокує перехоплення даних, але не DAS |
| Екранування кабелю | Низька | Блокує лише RF-Opto |
| Переміщення кабелю | Висока | DAS все одно чує |
| Знищення свого кабелю при відході | Середня | Кінцева, але ми встигнемо |

### 6.2 Наша стратегія

1. **Паралельний кабель** — безконтактний, невиявлюваний
2. **Підключення до обривків** — після артудару, противник не контролює
3. **Швидкий TAP** — за 3-5 хвилин, потім йдемо
4. **Аналіз DAS, а не трафіку** — не залежить від шифрування

## 7. Протокол операції

### 7.1 Підготовка

```
□ Розвіддані про розташування ворожих позицій
□ Карта з позначеними знайденими кабелями
□ Комплект інструментів (див. п. 5)
□ Зв'язок з Master Node (через TFN або WireGuard)
□ План відходу
□ Прикриття (вогнева група)
```

### 7.2 Вихід

```
□ Вихід у темний час
□ Рух по заздалегідь розвіданому маршруту
□ Мінімальний склад (2-3 особи)
□ Тепловізійна невидимість (накидки)
□ Радіомовчання (зв'язок лише через TFN за наявності)
```

### 7.3 Робота на точці

```
□ Виявлення кабелю
□ Визначення напрямку (до противника / від противника)
□ Підготовка кінця (стріпер → скол → коннектор)
□ Підключення OTDR
□ Зняття трасування (30-60 сек)
□ DAS-сканування (60-120 сек)
□ Відключення
□ Маскування місця роботи
```

### 7.4 Повернення

```
□ Відхід за планом
□ Передача даних на Master Node
□ Аналіз результатів
□ Документування (ISR-звіт)
□ Планування наступної операції
```

## 8. Правові та етичні аспекти

- Ця документація призначена для **оборонних цілей**
- Усі методи розвідки через TFN є **пасивними** — не створюють випромінювання
- Використання оптоволокна противника в зоні бойових дій відповідає **законам та звичаям війни**
- У мирний час застосування обмежено **законами про перехоплення комунікацій**
