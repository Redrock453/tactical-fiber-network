# Алгоритм обробки даних SpiderLink + RF Detector

Система працює в три етапи:

---

## Етап 1: Edge Collection (Термінал в окопі)

### 1.1 Оптичний моніторинг (DOM)

Зчитування параметрів з SFP-модуля через шину I2C:

- **RX_power** — приймана потужність
- **TX_power** — передавана потужність
- **SNR** — відношення сигнал/шум
- **temperature** — температура модуля

### 1.2 Первинна фільтрація

```
1. Видалення DC-компоненти (фільтр високих частот)
2. Нормалізація даних
3. Віконна функція Ханніна
```

### 1.3 Фреймінг

Нарізка потоку даних на сегменти по 100-500 мс для аналізу.

```
Сирі дані → [Фільтр] → [FFT] → [Спектр]
```

---

## Етап 2: Характеризація сигнатур (Edge Analysis)

### 2.1 Акустичний аналіз

Швидке перетворення Фур'є (FFT) для виділення низькочастотних вібрацій:

| Частота (Гц) | Джерело |
|--------------|--------|
| 1-3 | Крок людини |
| 2-5 | Біг |
| 8-15 | Колеса |
| 2-8 | Гусениці |

### 2.2 RF-аналіз

Пошук високочастотних аномалій у фазі сигналу:

| Частота | Джерело |
|---------|--------|
| 433 МГц | FPV дрони |
| 915 МГц | Дальні FPV |
| 1.2 ГГц | Телеметрія |
| 2.4 ГГц | RC керування |
| 5.8 ГГц | Відео |

---

## Етап 3: Класифікація (Cloud/DigitalOcean)

Передача векторів ознак на сервер, де нейромережа приймає рішення:

### Паттерни

```
Pattern A: Одиночні імпульси 1.5 Гц → ПІХОТА
Pattern B: Широкосмуговий шум → РЕБ
Pattern C: Гармонічні коливання 20-40 Гц → КОЛЕСНА ТЕХНІКА
Pattern D: Низькочастотні коливання 2-8 Гц → ГУСЕНИЧНА ТЕХНІКА
```

### Блок-схема

```
         ┌─────────────┐
         │   ВОЛОКНО   │
         └──────┬──────┘
                ▼
    ┌────────────────────┐
    │  SFP (DOM дані)  │
    └─────────┬──────────┘
              ▼
    ┌────────────────────┐
    │  High-pass фільтр  │  ← видалення шуму
    └─────────┬──────────┘
              ▼
    ┌────────────────────┐
    │  FFT (1024 точки) │  ← спектр
    └─────────┬──────────┘
              ▼
    ┌────────────────────┐
    │  Feature extraction│  ← вектор ознак
    └─────────┬──────────┘
              ▼
    ┌────────────────────┐
    │  Classifier (ML)    │  ← класифікація
    └─────────┬───────────┘
              ▼
    ┌────────────────────┐
    │  Telegram Alert    │  ← сповіщення
    └───────────────────┘
```

---

## Параметри системи

| Параметр | Значення |
|----------|----------|
| Частота дискретизації | 1000 Гц |
| Розмір вікна FFT | 1024 |
| Час оновлення | 1 сек |
| Поріг детекції | 0.15 |
| Затримка | < 100 мс |
