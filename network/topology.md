# TFN Network Topologies — від простого до mesh

## Огляд еволюції

```
v0.1  Point-to-Point   →  довести працездатність
v0.2  Ring (кільце)    →  витримує 1 обрив
v0.3  Partial Mesh     →  витримує 2+ обривів
v0.4  Full Mesh        →  максимальна живучість
```

---

## Топологія 1: Point-to-Point (v0.1)

```
[A] ═══════════════════ [B]

  Node A              Node B
  (окоп)              (КП / тил)
```

### Параметри
| Параметр | Значення |
|----------|----------|
| Вузли | 2 |
| Лінки | 1 |
| Обриви, що витримує | 0 |
| Складність | Мінімум |
| Routing | Не потрібен |

### Застосування
- Довести, що fiber-лінк працює
- Виміряти latency, throughput
- Перший тест DAS

### Налаштування
```bash
# Node A
ip addr add 192.168.10.1/24 dev eth0

# Node B
ip addr add 192.168.10.2/24 dev eth0

# Тест
ping 192.168.10.2
iperf3 -s   # на B
iperf3 -c 192.168.10.2  # на A
```

## Топологія 2: Chain (v0.1+)

```
[A] ═══ [B] ═══ [C] ═══ [D]

Лінійне з'єднання 4 вузлів
Потрібне routing на проміжних вузлах
```

### Параметри
| Параметр | Значення |
|----------|----------|
| Вузли | 3-8 |
| Лінки | N-1 |
| Обриви, що витримує | 0 |
| Max hops | N-1 |
| Складність | Низька |

### Застосування
- З'єднання позицій вздовж лінії фронту
- Простий relay між позиціями

### Налаштування (з IP forwarding)
```bash
# На проміжних вузлах (B, C)
sysctl -w net.ipv4.ip_forward=1

# Або через BATMAN-adv (краще):
ip link add bat0 type batadv
ip link set eth0 master bat0
ip link set eth1 master bat0
ip link set bat0 up
ip addr add 192.168.10.X/24 dev bat0
```

### Недоліки
- Один обрив = мережа розділяється
- Кожен hop додає latency
- Немає резервування

## Топологія 3: Ring / Кільце (v0.2) — РЕКОМЕНДОВАНА

```
    A ═══════ B
    ║         ║
    ║         ║
    D ═══════ C

Кожен вузол має 2 з'єднання
Будь-який обрив → мережа продовжує працювати
```

### Параметри
| Параметр | Значення |
|----------|----------|
| Вузли | 4+ |
| Лінки | N |
| Обриви, що витримує | **1** |
| Max hops | N/2 |
| Складність | Середня |

### Чому ring — перший крок
1. **Мінімум обладнання** — кожен вузол має рівно 2 порти
2. **Один обрив = працює** — найважливіша властивість
3. **Проста логіка** — не потрібен складний routing
4. **Легко тестувати** — розрізав кабель, перевірив зв'язок

### Налаштування
```bash
# Кожен вузол (2 fiber-порти)
modprobe batman-adv
ip link add bat0 type batadv
ip link set eth0 master bat0   # порт 1 (SFP #1)
ip link set eth1 master bat0   # порт 2 (SFP #2)
ip link set eth0 up
ip link set eth1 up
ip link set bat0 up
ip addr add 192.168.10.X/24 dev bat0
```

### Failover-тест
```bash
# 1. Перевірити маршрути
batctl o

# 2. Ping через ring
ping 192.168.10.2

# 3. Розрізати кабель A→B

# 4. Засікти час відновлення
# Очікувано: 3-5 секунд

# 5. Перевірити нові маршрути
batctl o  # A→D→C→B замість A→B
```

### Вимоги до обладнання
| Вузол | Потрібно |
|-------|----------|
| 2 Ethernet-порти | SFP + onboard, або 2× USB-Ethernet |
| Або | Banana Pi BPI-R3 (вбудований SFP) |
| Або | 1× медіаконвертер + 1× USB-Ethernet |

## Топологія 4: Partial Mesh (v0.3)

```
    A ═══════ B ═══════ C
    ║ ╲      ║ ╲      ║
    ║   ╲    ║   ╲    ║
    ║     ╲  ║     ╲  ║
    D ═══════ E ═══════ F

Додаткові діагональні зв'язки
2+ маршрутів між будь-якими вузлами
Витримує 2+ обривів
```

### Параметри
| Параметр | Значення |
|----------|----------|
| Вузли | 6+ |
| Лінки | 8-12 |
| Обриви, що витримує | **2+** |
| Max hops | 2-3 |
| Складність | Висока |

### Коли переходити
- Після успішного ring-тесту
- Коли є 6+ вузлів
- Коли потрібна живучість при артобстрілі

## Рекомендація

**Почати з Ring (v0.2)** — це найкращий баланс:
- Витримує 1 обрив (найчастіший сценарій)
- 4 вузли = реалістичний MVP
- Кожен вузол має тільки 2 порти
- BATMAN-adv працює "з коробки"
