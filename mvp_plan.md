# TFN MVP Plan — 3 дні до робочого прототипу

## Мета

Зібрати **працюючу fiber-лінію** довести що:
1. Кабель передає дані без RF
2. Лінія може працювати як сенсор (псевдо-DAS)
3. Все зібрано з доступних компонентів за $250

### Що робити

```bash
# 1. Зібрати вузли
RPi → Ethernet → Медіаконвертер → SFP → [Fiber] → SFP → Медіаконвертер → Ethernet → RPi

# 2. Встановити ОС
# Raspberry Pi OS Lite (64-bit)
# Увімкнути SSH

# 3. Налаштувати IP
# Node A:
ip addr add 192.168.10.1/24 dev eth0

# Node B:
ip addr add 192.168.10.2/24 dev eth0
```

### Тести

```bash

# Очікувано: < 1ms

# 2. Throughput

# Очікувано: 900+ Mbps

# 3. Відео стрім
# На A: ffmpeg -i video.mp4 -f mpegts udp://192.168.10.2:1234
# На B: ffplay udp://0.0.0.0:1234

# 4. Обрив кабелю
# Розрізати → зафіксувати час падіння лінії
# Відновити сплайсом → зафіксувати час відновлення
```

## День 2: Сенсорика (Pseudo-DAS)

### Що купити додатково

| Компонент | Ціна |
|-----------|------|
| ESP32-S3 DevKit | $10 |
| ADS1256 модуль | $12 |
| Piezo disc 27mm ×10 | $1 |
| Breadboard + дроти | $5 |
| **Разом** | **$28** |

### Що робити

1. Зібрати п'єзо + ADS1256 + ESP32
2. Підключити ESP32 до RPi через USB (serial)
3. Запустити reader script
4. **Записати сигнали:**

```bash
# Запис 30 секунд тиші
python sensing/record.py --duration 30 --label silence

# Запис кроків (5м від сенсора)
python sensing/record.py --duration 30 --label footsteps_5m

# Запис автомобіля (10м)
python sensing/record.py --duration 30 --label vehicle_10m
```

### Результат Дня 2

```
✅ П'єзо-сенсор знімає сигнал
✅ FFT показує різні частоти для різних цілей
✅ Збережено CSV файл для кожного типу
✅ Базова класифікація працює (rule-based)
```
